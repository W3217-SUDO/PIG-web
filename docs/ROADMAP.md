## 1. 上线目标的精确定义

> ⚠️ 一定要明确"上线"是什么,否则团队互相理解不一致。

**5/31 23:59 前,我们认为完成上线的标准是**:

### 1.1 微信小程序 v1.0.0
- [ ] 已通过微信审核(状态 = 「已发布」)
- [ ] 用户能完整走通:**进入 → 登录 → 浏览猪 → 下单 → 支付 → 看订单 → 看「我的猪」**
- [ ] 至少 5 头真实猪只数据上架(由运营手工 Navicat 入库)
- [ ] 至少 1 名内部测试用户完整跑通付款流(可用最小金额 ¥0.01 走通)

### 1.2 APP(降级目标)
- [ ] iOS:**TestFlight 内部测试可装**(不强求上架 App Store)
- [ ] Android:**APK 包可侧载安装**(可选:腾讯应用宝 / 华为提审中)

### 1.3 管理(妥协目标)
- [ ] 运营用 **Navicat 直连 RDS** 完成商家发布猪、改订单状态等操作
- [ ] 提供 ≥3 篇运营 SOP 文档(`docs/06-deployment/admin-sop.md`,新建)

### 1.4 不在 5/31 范围内的事(明确写下来,避免临时改口)
- 拼猪完整 v2(成员管理 / 权限隔离 / 踢出 / 退出)
- 实时直播(24h 推流 / 防盗链 / 回放)
- 喂养打卡(农户端写入)
- 健康档案写入
- 保险购买与理赔
- GUI 管理后台
- 商家发布猪自助流(由运营代发)
- 多农户接入(初期只有 1-2 个 demo 农户)

## 8. 升级 / 回滚预案

### 容量升级触发条件
- 日活 > 200 / 在线 > 50 → 升 8G 内存 + 独立 MySQL RDS
- DB QPS > 500 / 慢查询 > 1% → 加 RDS 只读实例

### 故障回滚
- 后端:pm2 reload 旧版本(`/opt/pig/releases/<prev-ts>/`)
- 小程序:微信公众平台 → 版本管理 → 回退线上版本
- DB schema:每次 migration 必须有 down(强制 review 点)
