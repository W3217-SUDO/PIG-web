# 产品定位 · 私人订猪

> **「认一头猪,过一个年。」** —— 城市家庭代养土猪平台。

---

## 一、我们做什么

用户在 App / 小程序 上**「认领」一头猪仔** → 川北山区农户代为饲养 **10–12 个月**(整整一年) → 腊月出栏,整猪 / 分割运到家。**用户全程通过 24h 直播 + 每餐打卡看见这头猪如何长大。**

简单说:
- **不是卖肉** —— 我们是替你养一头猪
- **不是养殖场** —— 是分散在川北山区的散养农户
- **不是预订** —— 是认领,这头猪从满月那天起就是你的

---

## 二、核心商业逻辑(差异化护城河)

### 传统卖肉商的痛点

| 角色 | 收入来源 | 内在动机 |
|---|---|---|
| 肉商 / 屠宰场 | 卖肉数量 × 单价 | 卖更多 / 长更快 / 重更重 → **催肥饲料 / 抗生素 / 注水** |
| 工厂养殖场 | 整体出栏量 | 缩短周期 → **激素 / 限制活动 / 高密度圈养** |
| 用户 | 想吃放心肉 | ❌ 信息不对称,无法监督 |

**这是结构性的利益对立**——不是因为人坏,是因为商业模式逼着商家这么干。

### 我们的解构

| 我们做的事 | 收入来源 | 内在动机 |
|---|---|---|
| 代养 + 平台服务 | 代养费 ¥10/天 + 餐食费 ¥10/天 | **猪是用户的,我们没动机催肥** |
| 给催肥饲料 = **害自己的客户** | — | 不会做,也没必要做 |

**关键洞察**:
> 商业模式决定立场。立场决定你能不能放心。
> **我们的盈利与「猪长多重多快」完全脱钩**——所以才能真做到放心猪肉。

## 三、四大核心卖点(产品文案、首页、品牌沟通的统一锚点)

### 1. 🏮 过年杀猪饭 · 家乡年味

> "村口大铁锅,亲戚邻居围着烧水、烫毛、灌香肠、喝刨汤——这才是中国人的过年。"

腊月杀年猪是中国人最朴素也最郑重的一场年。城里人有了空调和车,把这场年弄丢了。**我们把它还回去。**

文案抓手:**「认下的不只是一头猪,是一个回不去的过年。」**

### 2. 🌾 小时候的肉味道 · 慢养土猪

- 全部川北山区散养(广元 / 绵阳 / 德阳)
- **10–12 个月**自然生长(工厂猪 3–4 个月即出栏)
- 粗粮 + 蔬菜 + 山泉,无催肥饲料
- 出栏体重 150–180 斤,密度肉而非松垮的水肉

文案抓手:**「整整一年长成的猪,才是过年那种肉。」**

### 3. 🛡️ 不卖肉,只替你养 · 结构性安全

(见第二节"核心商业逻辑")

文案抓手:**「我们不是卖肉的,所以你才能放心。」**

### 4. 📹 全程可监控 · 不需要相信,你自己看

| 环节 | 用户能看到 |
|---|---|
| 圈舍 | 24h 直播 + 30 天回放 |
| 喂养 | 早 / 中 / 晚三餐 农户现场打卡(图片 + 食材) |
| 健康 | 每次体检 / 兽医 / 疫苗 自动写档 |
| 出栏 | 屠宰过程留档,冷链 GPS 追踪到家 |

文案抓手:**「从满月到上桌,每一天你都看得见。」**

## 四、拼猪 · 模式升级

### 用户洞察(2026 年 5 月,联合创始人提出)

> "一家未必吃得完一头 150 斤的猪。亲戚朋友拼一头,刚刚好。"

### 产品形态

| 档位 | 推荐场景 |
|---|---|
| 一家独养 | 大家庭 / 餐饮主理人 |
| **两家拼猪(推荐)** | 兄弟姐妹 / 一对邻居 |
| 三家拼猪 | 同学群 / 部门同事 |

### 流程

```
1. 主认领人下单一头猪
   ↓
2. 平台生成「专属分享码」
   ↓
3. 主认领人发给亲朋好友
   ↓
4. 受邀人凭码加入 → 同看一头猪(直播 / 喂养 / 档案全开放)
   ↓
5. 腊月出栏,平台只把货送到主认领人手里
```

### 平台边界(重要文案锚点)

> **「钱怎么 AA、谁出多少、谁来取、肉怎么分——这些你们自家商量。」**
> **平台不参与、不抽成、不仲裁。** 只负责把猪养好,把肉送到主认领人手里。

这是产品的**克制点**,也是反平台经济的**信任点**——我们不做"账本中介",因为亲戚朋友间的钱不能算得太清。

## 五、三端职责

| 端 | 用户 | 主要场景 |
|---|---|---|
| **C 端**(小程序 + APP + H5) | 认领人 / 拼猪受邀人 | 浏览、下单、看直播、拼猪、收货 |
| **代养人端**(暂复用 C 端) | 农户 | 每餐打卡、上传健康记录、收益查询 |
| **管理端**(Web 桌面) | 平台运营 | 用户 / 订单 / 猪只 / 农户 / 财务 / 监控 |

> 短期内,代养人端通过角色切换并入 C 端;农户量 > 50 时再独立 APP。

## 六、商业模型 / 单头猪经济(以黑土猪为例)

| 项 | 用户支付 | 平台收入 | 农户收入 |
|---|---|---|---|
| 猪仔基础费(认领时一次性) | ¥450(市价 30元/斤 × 15斤) | 0(转给农户) | ¥450 |
| 代养费(每日扣) | ¥10/天 × 365 = ¥3,650 | ¥3,650 × 20% = ¥730 | ¥2,920 |
| 餐食代购费(每日扣) | ¥10/天 × 365 = ¥3,650 | 0(转给农户进料) | ¥3,650 |
| 保险(可选,推荐) | ¥350 | ¥350 × 30% = ¥105 | 0(保司收) |
| 出栏物流(预估) | ¥200 | 0 | 0 |
| **合计单头** | **约 ¥8,300** | **约 ¥835** | **约 ¥7,020** |

- 平台毛利率 ≈ 10%
- 农户单头净收益 ≈ ¥3,000(扣除饲料采购成本 ¥3,650 后)
- 用户单头成本 ≈ ¥55/斤(出栏 150 斤),**与市场放心土猪肉持平**,但**确定性高 10 倍**

## 八、不做的事(产品克制清单)

- ❌ 不卖现成猪肉(没货源差异化)
- ❌ 不做生鲜电商(供应链与我们能力不匹配)
- ❌ 不做"拼猪账本中介"(分肉分钱用户自理)
- ❌ 不做"短期育肥"促销活动(违背慢养定位)
- ❌ 不接受未实名农户合作(可监控的前提是身份清楚)
- ❌ 不主动暴露农户个人微信 / 电话(防绕单)
