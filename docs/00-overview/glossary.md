# 项目术语表

> 所有写代码、写文档、写产品文案时,必须使用本文规定的标准术语。**不要自创同义词**——文案差异化是产品差异化的天然敌人。

---

## 业务核心术语

| 术语 | 英文 / 字段名 | 定义 |
|---|---|---|
| **认领人** | `user` / `claimer` | 在平台上认领一头(或一份)猪的用户 |
| **主认领人** | `primary_claimer` / `host` | 拼猪订单的发起人,持有猪的所有权 + 收货权 |
| **拼猪受邀人** | `co_claimer` / `guest` | 凭分享码加入拼猪订单的次要参与人 |
| **代养人** | `farmer` | 实际饲养猪的农户 |
| **猪只** | `pig` | 平台上一头被认领或可被认领的具体猪(有唯一编号) |
| **拼猪** | `share-claim` / `share` | 多人(2–3 家)共同认领一头猪的协作模式 |
| **分享码** | `share_code` | 主认领人生成的拼猪邀请码(6–8 位短码 + URL) |
| **年猪** | `nianzhu` / `year-pig` | 我们的产品本体——养满一年、为过年准备的土猪 |
| **杀猪饭** | `kill-pig-feast` | 腊月出栏时围炉吃刨汤的乡村仪式(平台情感锚点) |

## 状态术语

### 订单状态(`order.status`)

| 值 | 含义 |
|---|---|
| `pending_payment` | 已下单,未付猪仔基础费 |
| `active` | 认领生效,代养中 |
| `mature` | 达到出栏标准,等用户决策 |
| `slaughtering` | 已开始屠宰 |
| `shipping` | 已发货 |
| `delivered` | 已收货 |
| `cancelled` | 已取消(出栏前) |
| `refunded` | 已退款(意外死亡 + 保险触发) |

### 猪只状态(`pig.status`)

| 值 | 含义 |
|---|---|
| `available` | 未被认领,可供下单 |
| `claimed` | 已被认领,养殖中 |
| `quarantine` | 隔离观察(疾病 / 新入栏) |
| `mature` | 已达出栏 |
| `slaughtered` | 已屠宰 |
| `dead` | 死亡(走保险流程) |

### 拼猪关系状态(`share_member.status`)

| 值 | 含义 |
|---|---|
| `pending` | 已邀请但未接受(分享码点击,但未微信授权) |
| `joined` | 已加入,可看猪 |
| `left` | 自愿退出(钱由主认领人决定是否退) |
| `kicked` | 被主认领人移除 |

## 角色术语(`user.role`)

| 角色 | 权限 |
|---|---|
| `customer` | C 端用户(默认),可认领、拼猪、看自己的猪 |
| `farmer` | 代养人,可看 / 操作分配给自己的猪 |
| `admin` | 平台管理员,看所有数据 |
| `super_admin` | 超管,可修改密钥与配置 |

## 金额与计费术语

| 术语 | 含义 | 单位 |
|---|---|---|
| **猪仔基础费** | 认领时一次性支付,= 入栏市价 × 入栏体重 | 元 |
| **代养费** | 每日扣 ¥10,平台收 | 元/天 |
| **餐食费** | 每日扣 ¥10,平台代农户采购 | 元/天 |
| **保险费** | 一次性,选购 | 元/头 |
| **物流费** | 出栏后一次性,估算后预扣 | 元 |
| **平台抽成** | 平台从代养费抽 20%,保险抽 30% | 比例 |
| **钱包余额** | 用户预存款,每日自动扣代养费 + 餐食费 | 元 |

## 直播 / 监控术语

| 术语 | 含义 |
|---|---|
| **圈舍直播** | 猪圈摄像头的实时画面,24h 开放 |
| **喂养打卡** | 农户每餐(早/中/晚)拍照上传,系统记录 |
| **健康档案** | 体检 + 兽医 + 疫苗的累积记录 |
| **出栏直播** | 屠宰过程的直播或留档视频(可选订阅) |

## 技术术语

| 术语 | 含义 |
|---|---|
| **C 端 / Customer 端** | 用户使用的小程序 / APP / H5 |
| **农户端 / Farmer 端** | 农户使用的(短期复用 C 端,通过角色切换) |
| **管理端 / Admin** | 平台运营使用的桌面 Web 后台 |
| **代养系统** | 后端的核心业务模块,管理代养关系 / 喂养 / 健康 |

## 常见误用警示 ⚠️

| ❌ 不要写 | ✅ 正确写法 | 原因 |
|---|---|---|
| 客户 / 买家 / 消费者 | **认领人**(用户) | "买家"暗示我们是卖肉的,违背定位 |
| 商品 / 产品 / 商品 SKU | **猪只** | 一头猪不是 SKU,是一个独立个体 |
| 购买 / 下单 / 买猪 | **认领** | 下单流程 UI 也都写"认领" |
| 农场 / 工厂 / 基地 | **农户的猪圈** | 我们不做规模化养殖 |
| 工人 / 雇员 | **代养人 / 农户** | 农户是合作伙伴,不是雇员 |
| 拼团 / 团购 | **拼猪** | 团购暗示低价促销,拼猪是仪式分担 |
| 出货 / 发货 | **出栏 → 屠宰 → 配送** | 三个独立环节,各有用户感知 |
| 销售额 / 营收 | **服务费 + 餐食代购费** | 强化"我们不卖肉" |
