## 为什么单独建这个文件夹?

- `00-overview/` 讲**产品定位**(为什么做这个产品)
- `01-getting-started/` 讲**怎么搭建本地环境 + 服务器现状**(操作手册)
- `02-architecture/` 讲**怎么设计的**(架构原理)
- **`snapshot/`** 讲**现在跑到哪一步了**(状态快照,新人 5 分钟看完知道在哪)

跟 `ROADMAP.md`(战略) / `TASKS.md`(任务) 互补:
- 快照回答**「我们到哪儿了」**
- 路线图回答**「我们要去哪儿」**
- 任务清单回答**「下一步具体做什么」**

---

## 维护约定

- 这里的文档**不堆细节**,只写"概览级"事实
- 每次发版(`CHANGELOG.md` 加新版本号时)→ 顺手更新本文夹
- 内容跟其他 docs 冲突时,**以专门文档为准**(例如服务器细节以 `server-setup.md` 为权威),本快照只做指针
